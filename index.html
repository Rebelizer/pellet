Pellet

Making isomorphic apps easy!

Features

Imagine an SEO friendly AngularJS, a framework that unifies your client & server code into one codebase, a rich and powerful component system, a reactive programing model, and much more.

Say hello to little pellet!

  • Isomorphic

    Isomorphic is all the craze, Pellet simplifies your client and server codebase by making them the same. Now all you need to do is built your RIA app and Pellet make sure any page can be rendered on the server and serialized to the client.

  • React

    Pellet uses React to render your markup so it's blazing fast. It also makes event handling and user interactions soo much easier.

  • NodeJS

    We all love JavaScript and now we can run our code on both the server and client. With Pellet and NodeJS you no longer have to run multiple stacks or languages, you can just run one isomorphic environment.

  • Multilanguage

    If you have ever worked with internationalization you know it can be complex. Pellet makes it a cinch with built in pluralization, i18n, automatically handling formatting date/time, currency, numbers and has great build/deploy tools.

  • SEO

    Why do we have to sacrifice SEO for our RIA applications, or have to build complex fragment systems, or slow down the users first impression on first load? With Pellet, we do not need to sacrifice anything because any route is rendered as a static page, but can bootstrap into a full RIA application.

  • Reactive Programming

    Join all the cool kids and start using FRP to make your complex logic simple. Pellet offers a power full functional reactive programming library with a flexible event system.

  • Webpack

    As important as having great code you need a flexible and powerful build/deployment system. Pellet uses a manifest based system and webpack to make your code deployment easy. It creates optimized modular packages, multiple bundles for devices and environments, and transpiler your es6, jsx, CSS, etc into CDN ready bundles.

  • A/B/N Testing

    Imagine running your own multi-variant testing inside your single page application without twisting your brain. Pellet lets you A/B test anything from components, copy and logic. It is as simple as setting up a Pellet experiment or asking what that current variation is in your code.

  • Scalable Architecture

    Pellet was built to support VEVO a company that last year was responsible for 40+ billion videos watched globally and served out a ton of pages in multi languages and devices. Now Pellet is running VEVO site and VEVO has seen upward of 2x from SEO boost over AngularJS/BromBone and a 6x in page load times. Nice!

  • Cordova/Native

    Let's face it, the world is not only about WEB, but other devices, so Pellet is setup to support other devices like TV's, Consoles, and Mobile.

  • Flexible Date Architecture

    It's hard to manage the state of your data on the server and the client. So Pellet is built to help manage your data access layer's serialisation, rehydrating data on the client and data flow/syncretization.

  • Cutting Edge

    All browsers are not equal. Why should we have to support old browsers or be restricted to use new technologies like ES6. Pellet solves this problem, by polyfilling the browsers and lets you transpile and pre/post process your code. Bring the future of the web to you now.
